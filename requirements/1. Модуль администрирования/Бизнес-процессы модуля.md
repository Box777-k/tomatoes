| Номер  | Название бизнес-процесса                 | Краткое описание                                                                                                                                                                                                                                                                                   |
| ------ | ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| БП-1.1 | Первичная настройка системы              | Администратор создаёт первую организацию, настраивает базовые параметры учётной политики и создаёт пользователей с правами доступа. Этот процесс выполняется один раз при развёртывании системы. Включает определение типа организации (ЛПХ/ИП) и выбор модели разделения справочников.            |
| БП-1.2 | Добавление новой организации             | Администратор создаёт карточку новой организации (хозяйства или ИП), задаёт её тип и настройки учёта. Назначает права пользователям для работы в этой организации. Система автоматически обеспечивает изоляцию данных новой организации от существующих.                                           |
| БП-1.3 | Управление жизненным циклом пользователя | Администратор добавляет нового пользователя, назначает ему роли в контексте организаций, при необходимости блокирует или восстанавливает доступ. Все изменения прав фиксируются в журнале администрирования. При увольнении сотрудника учётная запись блокируется без удаления данных.             |
| БП-1.4 | Настройка ролей и прав доступа           | Администратор создаёт роли под конкретные должности (кладовщик, бухгалтер, руководитель), настраивает разрешения на уровне модулей и ключевых операций. Роли назначаются пользователям в разрезе организаций. Изменения ролей применяются немедленно ко всем пользователям с этой ролью.           |
| БП-1.5 | Переключение контекста работы            | Пользователь при входе в систему выбирает активную организацию из списка доступных ему. Все последующие операции, просмотр данных и отчёты автоматически фильтруются по выбранной организации. Переключение между организациями происходит через явное действие пользователя в интерфейсе.         |
| БП-1.6 | Настройка подразделений                  | Пользователь с правами администратора создаёт подразделения (участки, направления) внутри организации для аналитики и распределения ответственности. Подразделения используются в документах и отчётах. При реорганизации подразделение деактивируется, но история его использования сохраняется.  |
| БП-1.7 | Изменение учётной политики организации   | Администратор меняет базовые настройки организации, влияющие на правила учёта и проведения документов. Изменение имеет дату начала действия, система сохраняет историю всех изменений. При изменении критичных параметров система предупреждает о возможных последствиях для уже введённых данных. |
| БП-1.8 | Аудит административных действий          | Пользователь с правами аудитора просматривает журнал административных действий, фильтрует по периоду, типу операции или пользователю. Журнал показывает создание организаций, изменение прав, блокировки пользователей и изменения настроек. Записи журнала не подлежат удалению или изменению.    |
