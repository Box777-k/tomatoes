| **Базовые пользовательские истории**        |                                                      |                                                                                                                                                                                                                                                                                              |
| ПИ-1.1                                      | Создание первой организации                          | Как администратор, я хочу создать карточку моего ЛПХ/ИП при первом запуске системы, чтобы начать вести учёт. Я указываю название, тип организации (ЛПХ/ИП) и базовые реквизиты. Система создаёт организацию и автоматически назначает мне полные права администратора в её контексте.        |
| ПИ-1.2                                      | Добавление пользователя                              | Как владелец хозяйства, я хочу добавить сотрудника (кладовщика/бухгалтера) в систему и дать ему доступ к моей организации. Я ввожу его email, имя, назначаю роль и выбираю организации, в которых он может работать. Сотрудник получает приглашение и может войти в систему.                 |
| ПИ-1.3                                      | Переключение между организациями                     | Как пользователь с доступом к нескольким хозяйствам, я хочу легко переключаться между ними в интерфейсе. Я вижу список доступных мне организаций, выбираю нужную, и все данные на экране автоматически фильтруются по этой организации. Текущая активная организация всегда видна на экране. |
| ПИ-1.4                                      | Создание роли для кладовщика                         | Как администратор, я хочу создать роль "Кладовщик" с правами на просмотр и создание складских документов, но без доступа к финансам и бухгалтерии. Я выбираю разрешения из списка, сохраняю роль и назначаю её соответствующим пользователям. Система сразу применяет новые права.           |
| ПИ-1.5                                      | Просмотр прав пользователя                           | Как руководитель, я хочу видеть, какие права имеет конкретный сотрудник в каждой организации. Я открываю карточку пользователя и вижу список его ролей и разрешений в разрезе организаций. Это помогает мне контролировать безопасность доступа.                                             |
| ПИ-1.6                                      | Блокировка уволенного сотрудника                     | Как администратор, я хочу заблокировать доступ уволенного сотрудника без удаления его данных. Я нахожу пользователя в списке, нажимаю "Заблокировать", и он больше не может войти в систему. Все документы, созданные им ранее, остаются в системе с его именем.                             |
| ПИ-1.7                                      | Создание подразделения                               | Как владелец хозяйства с несколькими теплицами, я хочу создать подразделения для каждой теплицы, чтобы анализировать затраты отдельно. Я добавляю подразделения "Теплица №1", "Теплица №2" и при создании документов выбираю нужное подразделение для правильной аналитики.                  |
| ПИ-1.8                                      | Просмотр журнала администрирования                   | Как администратор, я хочу видеть, кто и когда менял настройки системы или права пользователей. Я открываю журнал, вижу хронологию всех административных действий с указанием даты, пользователя и типа изменений. Это помогает при расследовании инцидентов.                                 |
| **Альтернативные пользовательские истории** |                                                      |                                                                                                                                                                                                                                                                                              |
| ПИ-1.9                                      | Восстановление доступа заблокированного пользователя | Как администратор, я хочу восстановить доступ ранее заблокированного сотрудника, который вернулся на работу. Я нахожу его в списке заблокированных, нажимаю "Восстановить", и он снова может войти в систему с прежними правами.                                                             |
| ПИ-1.10                                     | Изменение настроек разделения справочников           | Как администратор, я хочу переключить режим работы справочника контрагентов с "общий для всех организаций" на "раздельный по организациям". Система предупреждает меня о последствиях, создаёт копии данных для каждой организации и фиксирует изменение с датой начала действия.            |
| ПИ-1.11                                     | Отказ в доступе при недостаточных правах             | Как пользователь с ролью "Кладовщик", я пытаюсь открыть раздел бухгалтерских проводок. Система показывает сообщение "Недостаточно прав для просмотра этого раздела" и фиксирует попытку доступа в журнале. Это защищает конфиденциальные данные.                                             |
| ПИ-1.12                                     | Деактивация подразделения                            | Как администратор, я хочу закрыть подразделение "Теплица №3", которое больше не используется, но сохранить историю. Я деактивирую подразделение, и оно исчезает из списка для выбора в новых документах, но остаётся в старых документах и отчётах за прошлые периоды.                       |
