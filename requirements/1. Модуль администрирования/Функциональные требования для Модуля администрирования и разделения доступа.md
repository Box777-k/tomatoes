| Номер | Краткое название функции приложения (web app) | Описание в 3-4 предложения                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1.1   | Создание организации                          | Пользователь создаёт карточку организации (юрлица/хозяйства) для ведения учёта в единой базе. Организация используется как верхний контекст всех данных: документы, склад, деньги, отчёты. Должна быть возможность выбрать тип организации (ЛПХ, ИП и т.п.) для дальнейших правил учёта. Доступ к организации ограничивается правами.                                                                                        |
| 1.2   | Просмотр и переключение активной организации  | Пользователь видит список доступных ему организаций и может переключать активную организацию в интерфейсе. Все экраны и операции должны автоматически фильтроваться по активной организации. Переключение должно быть быстрым и очевидным, чтобы исключить ввод данных “не туда”. Должна отображаться текущая активная организация.                                                                                          |
| 1.3   | Управление подразделениями организации        | Пользователь создаёт и редактирует подразделения/направления внутри организации для последующей аналитики и распределения ответственности. Подразделения могут использоваться в документах, движениях запасов и отчётах. Должна быть возможность отключать подразделение без удаления, чтобы сохранялась история. Иерархию делать необязательно, достаточно плоского списка.                                                 |
| 1.4   | Управление пользователями                     | Администратор добавляет пользователей в систему и назначает им доступ к конкретным организациям. Пользователь должен иметь возможность работать только в тех организациях, к которым ему выданы права. Должны поддерживаться блокировка и восстановление пользователя без удаления данных. Должна вестись история изменений прав.                                                                                            |
| 1.5   | Роли и права доступа                          | Администратор управляет ролями и разрешениями на уровне модулей и ключевых действий (просмотр, создание, проведение, отмена). Права должны применяться с учётом выбранной организации. Должна быть возможность назначать роли пользователю в разрезе организации. Система должна запрещать действия, на которые нет прав, и фиксировать попытки.                                                                             |
| 1.6   | Разграничение данных между организациями      | Система гарантирует изоляцию данных разных организаций в рамках одной базы. Запрещены операции, которые приводят к “смешиванию” документов, платежей или остатков разных организаций. Справочники могут быть общими или раздельными по настройке, но транзакционные данные должны быть строго по организации. Должны быть проверки целостности при сохранении и проведении документов.                                       |
| 1.7   | Журнал администрирования                      | Система ведёт журнал действий администраторов: создание организаций, назначение ролей, изменение прав, блокировки. Записи должны содержать дату/время, пользователя, действие и объект изменения. Журнал должен быть доступен только пользователям с соответствующими правами. Должен поддерживаться поиск и фильтрация.                                                                                                     |
| 1.8   | Настройки организации                         | Пользователь с правами администратора задаёт базовые настройки организации, влияющие на работу модулей. Настройки должны включать как минимум параметры учётной политики на уровне системы (например, использовать ли разделение справочников по организациям). Изменения настроек должны иметь дату начала действия и сохранять историю. Система должна предупреждать о рисках изменения настроек для уже введённых данных. |
