| Номер | Название бизнес-процесса | Краткое описание |
|-------|--------------------------|------------------|
| БП-2.1 | Создание и ведение контрагентов | Пользователь создаёт карточки поставщиков, покупателей и физических лиц с указанием реквизитов и контактных данных. Контрагенты используются во всех документах закупки, продажи и расчётов. Система контролирует уникальность контрагентов и предотвращает создание дублей. При необходимости контрагент архивируется без удаления истории операций. |
| БП-2.2 | Управление договорами с контрагентами | Пользователь создаёт договоры и привязывает их к контрагентам, указывает сроки действия и условия расчётов. Договоры используются в первичных документах и платежах для корректной идентификации обязательств. При истечении срока договор закрывается, но остаётся доступным для просмотра и отчётности. Система автоматически уведомляет об истекающих договорах. |
| БП-2.3 | Ведение номенклатуры продукции и товаров | Пользователь создаёт позиции номенклатуры с указанием типа (продукция, товар, материал, тара), единиц измерения и категории. Номенклатура используется во всех складских документах, продажах и производстве. Система не позволяет удалить номенклатуру, по которой были движения, только архивировать. Поддерживается массовое редактирование свойств номенклатуры. |
| БП-2.4 | Категоризация номенклатуры для аналитики | Пользователь создаёт иерархию категорий и распределяет номенклатуру по ним для удобства поиска и формирования отчётов. Категории используются в фильтрах документов, остатков и аналитических отчётах. При изменении категории номенклатуры история операций сохраняется с новой категорией в отчётах. Система запрещает удаление категорий, содержащих номенклатуру. |
| БП-2.5 | Настройка складской инфраструктуры | Пользователь создаёт карточки складов, точек продаж и промежуточных мест хранения с указанием ответственных лиц. Для крупных складов дополнительно создаются зоны и ячейки хранения. Склады используются во всех движениях запасов и формировании остатков. При реорганизации склад деактивируется, но история движений сохраняется. |
| БП-2.6 | Назначение материально ответственных лиц | Пользователь создаёт справочник МОЛ и назначает их на конкретные склады и точки продаж с указанием периода ответственности. МОЛ используются в документах перемещения, отгрузки и инвентаризации. При смене МОЛ проводится передача остатков с фиксацией акта приёма-передачи. Система формирует отчёты по остаткам и движениям в разрезе МОЛ. |
| БП-2.7 | Управление ценообразованием | Пользователь создаёт прайс-листы для разных каналов продаж (розница, опт, точки) и заполняет цены по номенклатуре. Прайсы имеют период действия и автоматически применяются в документах продажи. При изменении цен создаётся новая версия прайса с датой начала действия. Система сохраняет историю всех изменений цен для анализа ценовой политики. |
| БП-2.8 | Классификация расходов для управленческого учёта | Пользователь создаёт статьи расходов (топливо, корма, электричество, зарплата и т.п.) для управленческой аналитики. Статьи используются при регистрации платежей и распределении затрат по центрам ответственности. Система позволяет сопоставлять статьи расходов с бухгалтерскими счетами для интеграции управленческого и бухгалтерского учёта. При необходимости статья закрывается без удаления истории. |
| БП-2.9 | Настройка центров затрат и ответственности | Пользователь создаёт центры ответственности (теплицы, склады, точки продаж, логистика) для управленческого контроля. Центры используются при отражении расходов, амортизации и калькуляции себестоимости. Система автоматически распределяет затраты по центрам на основе первичных документов. Отчёты показывают расходы, доходы и маржинальность по каждому центру. |
| БП-2.10 | Массовое обновление справочных данных | Пользователь импортирует или массово редактирует данные справочников через загрузку файлов или групповое изменение. Система проверяет корректность данных, выявляет дубли и несоответствия. После проверки изменения применяются пакетно с фиксацией в журнале изменений. Это ускоряет первичную настройку системы и периодическую актуализацию данных. |
| БП-2.11 | Контроль актуальности справочных данных | Система автоматически отслеживает неиспользуемые элементы справочников и устаревшие данные (истёкшие договоры, неактивные контрагенты). Пользователь получает уведомления о необходимости актуализации. Формируются отчёты по качеству данных с рекомендациями по очистке и архивированию. Это поддерживает справочники в актуальном состоянии и упрощает работу. |
