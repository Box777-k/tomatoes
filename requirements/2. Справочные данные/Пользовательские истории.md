| **Базовые пользовательские истории**        |                                                    |                                                                                                                                                                                                                                                                                     |
| ПИ-2.1                                      | Добавление нового поставщика                       | Как менеджер по закупкам, я хочу создать карточку нового поставщика удобрений с указанием названия, ИНН и контактов. Я заполняю обязательные поля, система проверяет уникальность ИНН и сохраняет контрагента. Теперь я могу использовать его в документах закупки.                 |
| ПИ-2.2                                      | Создание договора с контрагентом                   | Как владелец хозяйства, я хочу зарегистрировать договор поставки с оптовым покупателем томатов. Я выбираю контрагента, указываю номер договора, срок действия и условия оплаты. Договор сохраняется и становится доступен при оформлении продаж этому покупателю.                   |
| ПИ-2.3                                      | Добавление новой позиции номенклатуры              | Как кладовщик, я хочу добавить в справочник новый сорт томатов "Черри красный". Я указываю название, выбираю категорию "Продукция собственного производства", единицу измерения "кг" и сохраняю. Теперь этот сорт доступен для использования в документах прихода и продажи.        |
| ПИ-2.4                                      | Создание категорий номенклатуры                    | Как владелец хозяйства, я хочу создать категории "Томаты", "Огурцы", "Зелень", "Услуги" для группировки номенклатуры. Я создаю категории, затем распределяю позиции номенклатуры по ним. Это упрощает поиск и формирование отчётов по группам продукции.                            |
| ПИ-2.5                                      | Регистрация нового склада                          | Как администратор, я хочу зарегистрировать склад "Теплица №2" с указанием ответственного кладовщика. Я создаю карточку склада, выбираю тип "Теплица" и назначаю МОЛ. Теперь склад доступен для выбора в документах движения товара.                                                 |
| ПИ-2.6                                      | Назначение МОЛ на склад                            | Как руководитель, я хочу назначить нового кладовщика Иванова ответственным за склад "Основной". Я создаю запись МОЛ с датой начала ответственности, предыдущий МОЛ автоматически завершает период. Система фиксирует передачу остатков для инвентаризации.                          |
| ПИ-2.7                                      | Создание прайс-листа для розницы                   | Как менеджер по продажам, я хочу создать розничный прайс-лист на летний сезон. Я создаю новый прайс, указываю период действия и заполняю цены по номенклатуре томатов. Прайс автоматически применяется в точках розничной продажи с указанной даты.                                 |
| ПИ-2.8                                      | Добавление статьи расходов                         | Как бухгалтер, я хочу создать статью расходов "Электроэнергия теплицы" для отслеживания этих затрат. Я создаю статью, указываю категорию "Коммунальные расходы" и при необходимости сопоставляю с бухгалтерским счётом. Статья становится доступна при регистрации платежей.        |
| ПИ-2.9                                      | Создание центра затрат                             | Как владелец с несколькими локациями с теплицами, я хочу создать центры затрат "Теплица №1", "Теплица №2", "Теплица №3" для раздельного учёта расходов. Я создаю центры и привязываю к ним склады. Система автоматически распределяет расходы по центрам при проведении документов. |
| ПИ-2.10                                     | Архивирование неактивного контрагента              | Как менеджер, я хочу архивировать поставщика, с которым больше не работаем. Я открываю карточку контрагента, нажимаю "Архивировать", он исчезает из списков выбора, но остаётся в старых документах. История операций с ним полностью сохраняется.                                  |
| ПИ-2.11                                     | Поиск номенклатуры по категории                    | Как кладовщик, я хочу быстро найти все позиции категории "Томаты" для инвентаризации. Я открываю справочник номенклатуры, выбираю фильтр по категории "Томаты", система показывает только нужные позиции. Это ускоряет работу с большим количеством номенклатуры.                   |

Я создаю два прайса: "Центр города" и "Рынок", привязываю их к соответствующим точкам. Система автоматически применяет нужный прайс при оформлении продажи.             |

Все новые операции автоматически идут под моей ответственностью.                                |

Я настраиваю в номенклатуре базовую единицу "кг" и дополнительную "ящик" с коэффициентом. Система автоматически пересчитывает количество при оформлении документов в разных единицах.                                             |
