| Номер | Краткое название функции приложения (web app) | Описание в 3-4 предложения                                                                                                                                                                                                                                                                                                                         |
| ----- | --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 2.1   | Ведение справочника контрагентов              | Пользователь создаёт и ведёт контрагентов: поставщиков, покупателей, подрядчиков и физлиц. Контрагент должен использоваться во всех документах закупки, продажи, услуг и расчётов. Должен поддерживаться поиск, фильтрация и предотвращение дублей. Контрагент может быть общим для всех организаций или отдельным, в зависимости от настройки.    |
| 2.2   | Ведение договоров                             | Пользователь создаёт договоры и привязывает их к контрагентам. Договор используется в первичных документах и платежах для корректной идентификации условий расчётов. Должна поддерживаться история договоров и возможность закрывать договор без удаления. Должны быть быстрые ссылки из операций на договор.                                      |
| 2.3   | Номенклатура продукции, товаров и материалов  | Пользователь ведёт единый справочник номенклатуры для производства, перепродажи и хозяйственных нужд. Номенклатура должна использоваться в складских движениях, первичных документах и отчётности. Должны поддерживаться типы номенклатуры (продукция, товар, материал, тара, топливо). Должна быть возможность архивировать позиции без удаления. |
| 2.4   | Категории номенклатуры                        | Пользователь создаёт категории для группировки номенклатуры и формирования отчётов. Категории используются для фильтрации, аналитики и настройки правил учёта. Система должна позволять менять категорию без потери истории операций. Должна быть предусмотрена защита от удаления категорий, уже используемых в данных.                           |
| 2.5   | Единицы измерения                             | Пользователь ведёт справочник единиц измерения (кг, шт, ящик, литр и т.п.). Единицы используются в номенклатуре и документах для корректного учёта количества. Должна быть поддержка базовой единицы и отображения кратности при необходимости. Изменение единицы для используемой номенклатуры должно быть ограничено правилами.                  |
| 2.6   | Склады и точки хранения                       | Пользователь создаёт склады, точки продаж и иные места хранения, используемые в учёте запасов. Склады применяются в движениях товара, остатках, резервах и инвентаризациях. Должны поддерживаться статусы активности и запрет удаления при наличии истории. Должна быть возможность ограничивать доступ пользователя к конкретным складам.         |
| 2.7   | Локации внутри склада                         | Пользователь при необходимости ведёт зоны/ячейки хранения внутри склада для точного контроля. Локации используются как дополнительная аналитика к складским движениям и остаткам. Модуль должен работать и без локаций, если они не нужны. Должно быть возможно отключить локации для конкретного склада.                                          |
| 2.8   | Материально ответственные лица                | Пользователь ведёт справочник МОЛ: кладовщики, курьеры, продавцы на точках. МОЛ используется в операциях перемещения и хранения для контроля ответственности. Должны поддерживаться назначения МОЛ на склад/точку и завершение назначения без удаления истории. Отчёты должны позволять видеть остатки по МОЛ.                                     |
| 2.9   | Прайсы и правила цен                          | Пользователь создаёт прайс-листы для розницы и опта и задаёт цены по номенклатуре. Прайсы используются при оформлении продаж и для контроля уценки. Должны поддерживаться даты действия и история изменений. Должна быть возможность разных прайсов для разных организаций и точек продаж.                                                         |
| 2.10  | Статьи расходов управленческого учёта         | Пользователь ведёт статьи расходов для управленческой аналитики и план-факт контроля. Статьи расходов используются в платежах, документах услуг и распределениях затрат. Система должна позволять сопоставлять статьи расходов с бухгалтерскими счетами, но не требовать этого обязательно. Должна быть поддержка закрытия статьи без удаления.    |
| 2.11  | Центры ответственности и затрат               | Пользователь создаёт центры ответственности (теплицы, склад, торговая точка, логистика и т.п.) для управленческого разреза. Они используются при отражении расходов, амортизации и распределении затрат. Должна быть возможность назначать центр по умолчанию для склада/точки. Отчёты должны показывать расходы и маржинальность по центрам.      |
