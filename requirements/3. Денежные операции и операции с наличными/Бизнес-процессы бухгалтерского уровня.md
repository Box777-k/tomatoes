# Бизнес-процессы бухгалтерского уровня - Финансы

Бухгалтерский уровень отвечает за формирование проводок, учёт расчётов с контрагентами и привязку к первичным документам.

| Номер | Название бизнес-процесса | Краткое описание |
|-------|--------------------------|------------------|
| БУ-3.1 | Генерация бухгалтерских проводок из операционных транзакций | Система автоматически формирует бухгалтерские проводки на основании операций движения денег из операционного уровня. Для каждой операции создаётся проводка с дебетом и кредитом счетов согласно настроенным правилам. Проводки содержат ссылку на исходную операционную транзакцию. |
| БУ-3.2 | Привязка платежей к первичным документам | Пользователь связывает операцию движения денег с одним или несколькими первичными документами (накладные, акты, счета). Указываются суммы зачёта по каждому документу. Система автоматически рассчитывает статус оплаты документа. Поддерживается частичная оплата и зачёт предоплаты. |
| БУ-3.3 | Учёт расчётов с контрагентами | Система ведёт учёт дебиторской и кредиторской задолженности по каждому контрагенту. Задолженность формируется на основании первичных документов и фактических платежей. Пользователь видит текущий остаток задолженности, историю взаиморасчётов, просроченные обязательства. |
| БУ-3.4 | Сверка расчётов с контрагентами | Пользователь проверяет соответствие начислений и фактических оплат по конкретному контрагенту за период. Система показывает документы начисления, платежи и текущий остаток задолженности. При обнаружении расхождений вносятся корректировки или уточняется привязка платежей. Результат оформляется актом сверки. |
| БУ-3.5 | Учёт целевого финансирования по грантам | Пользователь регистрирует поступления грантовых средств с указанием грантового соглашения и статьи бюджета. Расходы грантовых средств привязываются к разрешённым статьям с контролем лимитов. Система контролирует целевое использование. Формируются отчёты об освоении гранта. |
| БУ-3.6 | Контроль кассовых разрывов и ликвидности | Система анализирует текущие и прогнозируемые остатки денежных средств, предупреждает о рисках кассовых разрывов. Пользователь просматривает график платежей и поступлений, планирует движение денег. Формируются отчёты о динамике ликвидности. |
| БУ-3.7 | Закрытие периода и формирование отчётности | Пользователь закрывает учётный период после завершения всех операций. Система проверяет корректность проводок, формирует обороты по счетам, рассчитывает финансовый результат. После закрытия периода операции в нём становятся недоступными для изменения без специальных прав. |

## Особенности бухгалтерского уровня

- Использует данные из операционного уровня как источник
- Формирует проводки по плану счетов бухгалтерского учёта
- Связывает операции с контрагентами и документами
- Поддерживает закрытие периодов
- Формирует регламентированную отчётность
- Не изменяет данные операционного уровня

## Поток данных

Операционная транзакция (движение денег) → Бухгалтерская проводка (дебет/кредит) → Привязка к документу → Учёт расчётов с контрагентом
