
| Номер | Краткое название функции приложения (web app) | Описание в 3–4 предложения                                                                                                                                                                                                                                                                                   |
| ----- | --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 3.1   | Банковские счета организации                  | Пользователь ведёт банковские счета в разрезе организаций. Счёт используется во всех безналичных платежах и для формирования остатков денежных средств. Должна поддерживаться возможность закрытия счёта без удаления истории операций. Один счёт не может принадлежать нескольким организациям.             |
| 3.2   | Кассы организации                             | Пользователь создаёт кассы для учёта наличных денежных средств. Касса привязывается к организации и может быть связана с конкретной точкой продаж или МОЛ. Должен вестись текущий остаток и история операций. Удаление кассы при наличии операций запрещено.                                                 |
| 3.3   | Учёт поступлений денежных средств             | Пользователь отражает поступление денег в кассу или на расчётный счёт. Поступление должно иметь источник: контрагент, грант, собственные средства, прочий доход. Операция используется для формирования остатков и дальнейших бухгалтерских проводок. Должна поддерживаться привязка к первичному документу. |
| 3.4   | Учёт списаний денежных средств                | Пользователь отражает расход денежных средств с указанием назначения платежа. Списание связывается с контрагентом, статьёй расходов и при необходимости с грантом. Операция уменьшает остаток денежных средств и используется в бухучёте. Система должна проверять наличие достаточного остатка.             |
| 3.5   | Расчёты с контрагентами                       | Система ведёт учёт задолженности по каждому контрагенту в разрезе организаций. Данные формируются на основе начислений и фактических оплат. Пользователь должен видеть текущую дебиторскую и кредиторскую задолженность. Должны быть отчёты по просроченным обязательствам.                                  |
| 3.6   | Привязка платежей к документам                | Пользователь может связывать платёж с одним или несколькими первичными документами. Это позволяет контролировать оплату закупок, продаж и услуг. Система должна отображать статус оплаты документа. Частичные оплаты должны поддерживаться.                                                                  |
| 3.7   | Источники финансирования                      | Пользователь указывает источник средств для каждого поступления или расхода. Источник используется для управленческого контроля и отчётности, включая гранты и собственные средства. Один платёж может быть связан только с одним источником. История источников должна сохраняться.                         |
| 3.8   | Денежные остатки и контроль                   | Система автоматически рассчитывает остатки по кассам и банковским счетам. Пользователь должен видеть остатки на любую дату. Должны быть предупреждения о кассовых разрывах и отрицательных остатках. Остатки используются в отчётах и при проверках операций.                                                |
| 3.9   | Журнал денежных операций                      | Система ведёт единый журнал всех денежных операций. Журнал должен поддерживать фильтрацию по организации, счёту, кассе, контрагенту и периоду. Записи журнала не могут удаляться, только аннулироваться корректирующими операциями. Журнал используется для контроля и аудита.                               |
