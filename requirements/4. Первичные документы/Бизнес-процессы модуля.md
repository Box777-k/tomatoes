| Номер | Название бизнес-процесса | Краткое описание |
|-------|--------------------------|------------------|
| БП-4.1 | Создание и заполнение первичного документа | Пользователь создаёт документ выбранного типа (поступление товара, реализация, акт услуг, перемещение, списание) в черновике. Заполняет обязательные реквизиты: дату, контрагента, склад, добавляет строки с номенклатурой, количеством и ценами. Система автоматически рассчитывает суммы и итоги. Документ в черновике можно редактировать и удалять без ограничений. |
| БП-4.2 | Проведение документа и формирование движений | Пользователь проводит документ, система выполняет проверки: наличие обязательных полей, достаточность остатков, корректность счетов. При успешной проверке создаются движения: складские (приход/расход/перемещение), денежные (если задана оплата) и бухгалтерские проводки. Документ переходит в статус "Проведён" и становится недоступен для прямого редактирования. Все движения атомарны - либо проводятся все, либо ни одно. |
| БП-4.3 | Отмена проведения и корректировка документа | Пользователь с правами отменяет проведение документа, система проверяет возможность отмены (нет зависимых проведённых документов, период не закрыт). При отмене все созданные движения удаляются, документ возвращается в черновик. Пользователь корректирует данные и повторно проводит документ. Система фиксирует в истории факт отмены проведения с указанием причины. |
| БП-4.4 | Привязка документа к платежам | Пользователь связывает первичный документ с денежными операциями для контроля оплаты. Документ может быть связан с предоплатой (платёж до документа), постоплатой (платёж после) или частичными платежами. Система автоматически рассчитывает статус оплаты: "Не оплачен", "Частично оплачен", "Полностью оплачен". Привязка используется для формирования задолженности контрагентов. |
| БП-4.5 | Прикрепление сканов и подтверждающих файлов | Пользователь прикрепляет к документу сканы накладных, актов, счетов, фотографии товара или чеков. Файлы хранятся в привязке к документу и доступны для просмотра пользователям с правами. Это обеспечивает документальное подтверждение операций, особенно важно для грантовой отчётности. Система контролирует размер и формат файлов, ведёт версионность при замене. |
| БП-4.6 | Копирование документа для повторяющихся операций | Пользователь копирует существующий документ для быстрого создания аналогичного (например, еженедельная поставка от одного поставщика). Система создаёт новый черновик с заполненными данными исходного документа, но новым номером и текущей датой. Пользователь корректирует количество/цены и проводит новый документ. Это экономит время на рутинных операциях. |
| БП-4.7 | Формирование и печать документа | Пользователь формирует печатную форму документа по шаблону (накладная, товарная накладная ТОРГ-12, акт, счёт). Шаблон заполняется данными документа, реквизитами организации и контрагента. Печатная форма может быть выгружена в PDF или распечатана напрямую. Система сохраняет факт формирования печатной формы с датой и временем для аудита. |
| БП-4.8 | Поиск и фильтрация документов в журнале | Пользователь открывает журнал документов и применяет фильтры: по типу, периоду, контрагенту, складу, статусу проведения, статусу оплаты. Система быстро находит нужные документы даже при большом объёме данных. Фильтры можно сохранять как избранные для быстрого доступа. Журнал показывает ключевые данные: дату, номер, контрагента, сумму, статусы. |
| БП-4.9 | Массовое проведение документов | Пользователь выбирает несколько черновиков в журнале и выполняет массовое проведение. Система последовательно проводит каждый документ, фиксирует успешные и неудачные операции. По завершении показывает отчёт: сколько проведено успешно, сколько с ошибками и по каким причинам. Это ускоряет обработку большого количества однотипных документов. |
| БП-4.10 | Контроль целостности документооборота | Система автоматически контролирует корректность документооборота: поступление товара перед продажей, отсутствие отрицательных остатков, соответствие проводок принципу двойной записи. При попытке некорректной операции документ не проводится с понятным сообщением об ошибке. Система предупреждает о рисках (например, продажа по цене ниже себестоимости). Это обеспечивает качество учётных данных. |
