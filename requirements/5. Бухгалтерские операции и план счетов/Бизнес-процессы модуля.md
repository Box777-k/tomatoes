| Номер | Название бизнес-процесса | Краткое описание |
|-------|--------------------------|------------------|
| БП-5.1 | Настройка и адаптация плана счетов | Пользователь загружает типовой план счетов или создаёт собственный с учётом специфики бизнеса и формы налогообложения. План счетов включает счета первого порядка, субсчета и их иерархию. Для каждого счёта указываются тип (активный/пассивный/активно-пассивный), обязательные аналитики и правила использования. Изменения плана счетов версионируются с датой начала действия. |
| БП-5.2 | Настройка аналитических измерений | Пользователь определяет виды аналитик, необходимых для детализации учёта (склад, номенклатура, контрагент, договор, МОЛ, центр затрат, проект). Для каждого счёта настраивается, какие аналитики обязательны, а какие опциональны. Система автоматически контролирует заполнение обязательных аналитик при формировании проводок. Аналитики используются в оборотно-сальдовых ведомостях и отчётах. |
| БП-5.3 | Автоматическое формирование проводок при проведении документов | Система автоматически создаёт бухгалтерские проводки при проведении первичных документов согласно настроенным правилам. Каждый тип документа имеет шаблон проводок, который заполняется данными из документа (суммы, аналитики, счета). Все проводки формируются атомарно вместе со складскими и денежными движениями. Проводки всегда соответствуют принципу двойной записи: сумма по дебету равна сумме по кредиту. |
| БП-5.4 | Ручной ввод корректировочных проводок | Бухгалтер с соответствующими правами вводит ручные бухгалтерские операции для корректировок, переоценок и других операций, не имеющих автоматизированных документов. Каждая ручная операция обязательно имеет описание экономического смысла и обоснование. Система контролирует равенство дебета и кредита, корректность использования счётов и заполнение аналитик. Ручные проводки помечаются специальным признаком для аудита. |
| БП-5.5 | Просмотр и анализ журнала проводок | Пользователь просматривает журнал всех бухгалтерских операций с фильтрацией по периоду, счетам, контрагентам и документам-основаниям. Каждая проводка содержит ссылку на первичный документ для прослеживаемости. Журнал показывает дату, номер операции, дебет, кредит, суммы, аналитики и описание. Из журнала можно перейти к исходному документу или детализировать по аналитикам. |
| БП-5.6 | Формирование оборотно-сальдовой ведомости | Система автоматически рассчитывает оборотно-сальдовую ведомость по счетам за выбранный период. Отчёт показывает сальдо на начало, обороты по дебету и кредиту, сальдо на конец по каждому счёту. Поддерживается детализация до субсчетов и аналитик (ОСВ по счёту 60 с детализацией по контрагентам). Система контролирует баланс: итоговые дебетовые и кредитовые обороты должны быть равны. |
| БП-5.7 | Закрытие бухгалтерского периода | Бухгалтер выполняет регламентные операции конца месяца: начисление амортизации, закрытие счетов затрат, определение финансового результата. После проверки корректности данных период закрывается. В закрытом периоде запрещены изменения документов и проводок без специальных прав "Отмена закрытия периода". Статус закрытия периода явно отображается в интерфейсе. Закрытие периода обратимо при обнаружении ошибок. |
| БП-5.8 | Проверка корректности бухгалтерского учёта | Система автоматически выполняет контрольные проверки: равенство дебета и кредита во всех операциях, отсутствие проводок без первичных документов, корректность использования счетов, заполнение обязательных аналитик. При обнаружении ошибок формируется отчёт с указанием некорректных операций и рекомендациями по исправлению. Контроль выполняется автоматически при попытке закрытия периода и может быть запущен вручную в любое время. |
| БП-5.9 | Настройка правил автоматического формирования проводок | Администратор или главный бухгалтер настраивает правила формирования проводок для каждого типа документа (поступление, реализация, оплата и т.д.). Правило включает: какие счета дебетуются и кредитуются, откуда берутся суммы и аналитики, условия применения правила. Правила версионируются, изменения применяются с указанной даты. Это позволяет адаптировать систему под изменения учётной политики без программирования. |
| БП-5.10 | Формирование бухгалтерской отчётности | Система автоматически формирует бухгалтерский баланс (форма №1), отчёт о финансовых результатах (форма №2) и другие формы отчётности на основе данных проводок. Отчёты строятся по данным закрытых периодов с группировкой счетов по разделам баланса. Поддерживается сравнительный анализ отчётности за разные периоды. Отчёты можно выгружать в стандартных форматах для предоставления в контролирующие органы. |
