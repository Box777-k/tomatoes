| Номер | Краткое название функции приложения (web app) | Описание в 3–4 предложения                                                                                                                                                                                                                                                                                |
| ----- | --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 5.1   | План счетов                                   | Система хранит план счетов бухгалтерского учёта, используемый организацией. План счетов может быть типовым и адаптированным под конкретную организацию. Счета используются во всех бухгалтерских проводках. Изменения плана счетов должны сохранять историю и не ломать существующие данные.              |
| 5.2   | Субсчета и структура счетов                   | Пользователь управляет субсчетами и их иерархией в рамках счёта. Субсчета используются для детализации учёта (например, 01.01, 01.03, 08.03). Система должна контролировать корректность использования субсчетов в проводках. Удаление используемых субсчетов запрещено.                                  |
| 5.3   | Аналитики бухгалтерского учёта                | Пользователь настраивает виды аналитик (склад, номенклатура, МОЛ, договор, центр затрат и т.п.). Аналитики могут быть обязательными или опциональными для конкретных счетов. Система должна проверять заполнение аналитик при формировании проводок. Аналитики используются для отчётности и расшифровок. |
| 5.4   | Журнал хозяйственных операций                 | Система ведёт журнал бухгалтерских операций как «шапок» записей. Каждая операция имеет дату, организацию и описание экономического смысла. Операция может включать одну или несколько проводок. Журнал используется для контроля и аудита.                                                                |
| 5.5   | Бухгалтерские проводки                        | Система формирует проводки по принципу двойной записи (дебет/кредит). Проводки могут создаваться автоматически из документов или вручную пользователем с правами бухгалтера. Суммы по дебету и кредиту должны быть равны в рамках операции. Ручные проводки должны быть помечены как корректировочные.    |
| 5.6   | Связь проводок с документами                  | Каждая проводка должна быть связана с первичным документом или корректировочной операцией. Пользователь должен видеть, из какого документа возникла проводка. Это обеспечивает прослеживаемость и обоснованность учёта. Удаление документа не должно оставлять «висящих» проводок.                        |
| 5.7   | Закрытие бухгалтерского периода               | Пользователь выполняет закрытие месяца или периода. После закрытия запрещено изменение документов и проводок в этом периоде без специальных прав. Система должна поддерживать повторное закрытие при корректировках. Статус периода должен быть явно виден пользователю.                                  |
| 5.8   | Оборотно-сальдовая ведомость                  | Система формирует оборотно-сальдовую ведомость по счетам и аналитикам. Отчёт должен поддерживать детализацию до проводки. Пользователь выбирает период, организацию и фильтры. Данные отчёта должны сходиться с журналом проводок.                                                                        |
| 5.9   | Контроль корректности учёта                   | Система автоматически проверяет базовые бухгалтерские правила: равенство дебета и кредита, заполненность аналитик, допустимость счетов. При ошибках проведение операций должно блокироваться. Пользователь должен видеть понятные сообщения о причинах. Контроль обязателен для всех организаций.         |
| 5.10  | Корректировки и ручные операции               | Пользователь с соответствующими правами может вводить корректировочные бухгалтерские операции. Такие операции должны быть явно помечены и иметь обязательное описание причины. Корректировки не должны разрушать связь с отчётностью. История корректировок должна сохраняться.                           |
