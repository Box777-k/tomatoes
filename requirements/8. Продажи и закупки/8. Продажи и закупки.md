| Номер | Краткое название функции приложения (web app) | Описание в 3–4 предложения                                                                                                                                                                                                                                                                                       |
| ----- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 8.1   | Заказ покупателя                              | Пользователь оформляет заказ покупателя с указанием номенклатуры, количества, цены и точки отгрузки. Заказ используется для планирования отгрузок и резервирования товара. Заказ имеет статусы жизненного цикла (создан, подтверждён, выполнен, отменён). Заказ сам по себе не формирует бухгалтерских проводок. |
| 8.2   | Резервирование товара под заказ               | Система резервирует товар на складе или точке продаж под конкретный заказ. Зарезервированный товар исключается из доступного остатка, но остаётся на балансе. Резерв может быть частичным и иметь срок действия. История резервов должна сохраняться.                                                            |
| 8.3   | Реализация продукции и товаров                | Пользователь оформляет документ реализации для продажи продукции или товаров. Реализация формирует складское выбытие, выручку и себестоимость. Документ должен поддерживать розницу и опт. Реализация может быть связана с заказом покупателя.                                                                   |
| 8.4   | Возвраты от покупателей                       | Пользователь оформляет возврат проданной продукции или товаров. Операция корректирует складские остатки, выручку и расчёты с покупателем. Возврат всегда должен ссылаться на исходную реализацию. Должны поддерживаться частичные возвраты.                                                                      |
| 8.5   | Уценка и изменение цены                       | Пользователь фиксирует уценку продукции или товаров. Уценка используется для управленческого и бухгалтерского учёта потерь маржи. Операция не меняет количество, но влияет на финансовый результат. Причина уценки должна быть зафиксирована.                                                                    |
| 8.6   | Заказ поставщику                              | Пользователь оформляет заказ поставщику для планирования закупок. Заказ используется для контроля сроков и объёмов поставки. Заказ не формирует бухгалтерских проводок до фактического поступления. Должна поддерживаться связь с договором.                                                                     |
| 8.7   | Поступление от поставщика                     | Пользователь отражает поступление товаров, материалов или услуг от поставщика. Операция формирует складские остатки и расчёты с контрагентом. Поступление может быть частичным по отношению к заказу. Документ является основанием для бухгалтерского учёта.                                                     |
| 8.8   | Возвраты поставщику                           | Пользователь оформляет возврат поставщику некачественной или лишней продукции. Возврат корректирует складские остатки и расчёты. Операция должна быть связана с исходным поступлением. Причина возврата фиксируется.                                                                                             |
| 8.9   | Контроль взаиморасчётов                       | Система отображает состояние расчётов по каждому покупателю и поставщику. Пользователь видит оплаченные, неоплаченные и частично оплаченные документы. Данные используются для контроля задолженности. Отчёты формируются по организации и периоду.                                                              |
| 8.10  | Отчёты по продажам и закупкам                 | Система формирует отчёты по объёмам продаж, закупок, марже и оборачиваемости. Отчёты доступны по номенклатуре, складам, точкам продаж и контрагентам. Данные должны сходиться с бухгалтерским и складским учётом. Отчёты используются для управленческих решений.                                                |
