| Номер | Краткое название функции приложения (web app) | Описание в 3–4 предложения                                                                                                                                                                                                                                                                                                                   |
| ----- | --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 9.1   | Создание карточки гранта                      | Пользователь создаёт грант как отдельный источник целевого финансирования. Грант привязывается к организации и имеет срок действия. Грант используется для маркировки поступлений и расходов. Должна поддерживаться архивация гранта без удаления истории.                                                                                   |
| 9.2   | Условия и ограничения гранта                  | Пользователь фиксирует условия гранта: разрешённые направления расходов и ограничения по срокам. Эти условия используются для контроля при привязке платежей. Система должна предупреждать, если расход не соответствует условиям. Изменения условий должны сохранять историю.                                                               |
| 9.3   | Статьи бюджета гранта                         | Пользователь создаёт статьи бюджета гранта (например: оборудование, материалы, услуги, строительство). Статьи используются для планирования и последующей отчётности. Должны поддерживаться лимиты по сумме на статью. Удаление статьи при наличии операций запрещено.                                                                       |
| 9.4   | Учёт поступлений по гранту                    | Пользователь отражает поступление денег по гранту и связывает его с конкретным грантом. Поступление должно попадать в общий модуль денег, но иметь отметку гранта. Система должна показывать доступный остаток гранта. Должна поддерживаться частичная выдача/поступление траншами.                                                          |
| 9.5   | Привязка расходов к гранту                    | Пользователь при создании расхода может указать грант и статью бюджета. Система должна контролировать остаток лимита по статье и общий остаток гранта. При нарушении лимитов система должна блокировать проведение или требовать подтверждения роли администратора. Привязка должна сохраняться для отчётности и аудита.                     |
| 9.6   | Смешанное финансирование расходов             | Пользователь должен иметь возможность разделить один расход на несколько источников финансирования (грант + собственные средства). Система фиксирует доли и формирует корректные отчёты по гранту. Должны поддерживаться частичные оплаты и распределение по оплатам. Нельзя допускать, чтобы сумма распределения превышала сумму платежа.   |
| 9.7   | Контроль целевого использования               | Система проверяет соответствие расходов правилам гранта по статье, сроку и типу операции. Пользователь видит список нарушений и причины. Должна быть возможность зафиксировать служебное обоснование и решение ответственного лица. Нарушения должны попадать в отчёт по рискам гранта.                                                      |
| 9.8   | Отчёт по гранту «план/факт»                   | Система формирует отчёт по гранту: бюджет по статьям, факт расходов, остаток и отклонения. Отчёт должен работать на выбранную дату и за период. Должны быть фильтры по организации и гранту. Данные должны сходиться с журналом денежных операций и привязанными документами.                                                                |
| 9.9   | Реестр документов по гранту                   | Система показывает список всех первичных документов и платежей, связанных с грантом. Реестр используется для подготовки пакета отчётности грантодателю. Должны быть фильтры по статье, контрагенту и периоду. Из реестра должны открываться связанные документы и платежи.                                                                   |
| 9.10  | Экспорт отчётности по гранту                  | Пользователь выгружает отчёт и реестр документов в файл для предоставления грантодателю. Формат выгрузки должен поддерживать как минимум табличный вид и комплект подтверждающих документов (ссылки/идентификаторы). Экспорт должен фиксировать дату формирования и автора. Должна быть возможность повторной выгрузки без изменения данных. |
